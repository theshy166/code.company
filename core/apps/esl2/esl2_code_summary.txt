1. common_service.c
   - 路径: core/apps/esl2/ctrl/common_service.c
   - 用途: 应用初始化时创建必要的资源目录
   - 实现逻辑:
     在应用启动时调用 common_service_init()，依次创建三个目录:
       - 图片下载目录 (k_path_img_prefix)
       - JSON数据目录 (k_path_json_prefix)
       - 开机Logo目录 (k_path_logo_prefix)
     使用 transform_path() 转换路径后，调用 create_directory() 创建。
   - 核心函数: common_service_init()

2. esl_file.c
   - 路径: core/apps/esl2/ctrl/esl_file.c
   - 用途: 本地文件读写操作，包括JSON文件的保存、读取、删除
   - 实现逻辑:
     - _read_json_file(): 读取JSON文件内容到字符串
     - _save_json_to_file(): 将JSON字符串写入文件（先删旧文件再写新文件）
     - _els_base_model_to_json(): 将 esl_base_model_t 结构体序列化为JSON字符串
     - save_data_thread(): 在子线程中将UI模型数据保存到本地JSON文件
     - query_esl_base_model_local(): 从本地文件读取并反序列化UI模型
     - delete_local_data(): 删除本地保存的模型数据文件
     - delete_local_logo_image(): 删除开机Logo图片文件
     - _check_and_delete_file(): 检查文件是否存在，存在则删除
   - 核心函数: save_data_thread(), query_esl_base_model_local(), delete_local_data()

3. esl_message_queue.c
   - 路径: core/apps/esl2/ctrl/esl_message_queue.c
   - 用途: OTA消息队列管理，处理批量设备升级消息
   - 实现逻辑:
     - 使用栈结构 (ota_stack_t) 存储OTA消息，最大支持200条
     - handle_ota() 线程: 循环处理栈中的消息，每次最多处理3条(OTA_BATCH_SIZE)
       - 当栈为空时通过条件变量 pthread_cond_wait() 阻塞等待
       - 收到信号后sleep 5秒收集消息，然后批量处理
       - 处理完一批后如果还有剩余，sleep 60秒(MESSAGE_TIMEOUT)再继续
     - push_value(): 将消息深拷贝后压入栈，发送条件变量信号唤醒处理线程
     - _create_send_socket_slave(): 从QUA消息队列消费消息，解析JSON判断消息类型:
       - dt=10且type="version_update"时使用信号量限流（30秒超时）
       - 最终调用 create_send_socket() 发送TCP消息
     - msg_queue_init(): 初始化消息队列和OTA处理线程
     - post_msg_to_queue(): 向QUA消息队列投递消息
   - 核心函数: handle_ota(), push_value(), _create_send_socket_slave(), msg_queue_init()

4. esl_service.c
   - 路径: core/apps/esl2/ctrl/esl_service.c
   - 用途: HTTP通信服务，与服务器交互获取配置、内容、上报状态等
   - 实现逻辑:
     - 定义多个API URL模板 (KbaseUrl_CONFIG/ONLINE/MSGACK/CONTENT等)
     - _post_remote_url_with_form_only_esl(): 核心HTTP POST函数，使用libcurl
       - 设置 multipart/form-data 格式
       - 包含 appid, deviceno, type, sign 等必填字段
       - 可选字段: protocol_type, local_ip, device_list, time, version_no, push_id, push_type
       - 使用 _write_data() 回调接收响应数据
       - 设置10秒连接超时，30秒总超时
     - query_push_server(): 查询推送服务器配置，解析返回的JSON获取:
       - MQTT连接信息 (host, port, username, password, client_id, topic)
       - 固件更新信息 (update_version, update_url, update_file_md5)
       - Logo图片信息 (boot_logo_url, bind_logo_url)
       - 组播信息 (master, group_id)
     - parse_product_template(): 解析产品模板JSON，构建 esl_show_model_t 结构体
       - 解析模板名称、分辨率、类型、尺寸
       - 递归解析 child_list 子元素（位置、尺寸、角度、文本样式等）
       - 解析 desc 描述信息（颜色、字体、对齐方式等）
       - 解析 extension 扩展属性
   - 核心函数: query_push_server(), parse_product_template(), _post_remote_url_with_form_only_esl()

5. esl_socket_service.c
   - 路径: core/apps/esl2/ctrl/esl_socket_service.c
   - 用途: TCP Socket通信服务，设备间点对点通信
   - 实现逻辑:
     - 使用 hashmap 管理多个映射表:
       - map: 主Socket映射
       - check_life_map: 心跳检测映射
       - group_map_1/group_map_2: 组播分组映射（屏1和屏2）
       - push_id_map_1/2/3: 消息ID映射
     - socket_listener(): TCP监听线程，接受客户端连接
       - 动态缓冲区接收完整消息
       - 解析消息模型 (esl_message_model_t)
       - 根据消息类型(type)处理:
         - EslMessageTypeCONTENT: 内容更新，包括模板解析、资源下载、UI刷新
         - EslMessageTypeRESTRAT: 设备重启
       - 支持解绑操作 (unbind=1 时删除数据和UI)
     - process_server_response(): 处理服务器响应，根据type值释放信号量
     - send_response(): 向组内设备发送响应消息
     - create_send_socket_slave(): 创建TCP连接向从设备发送数据
     - push_msgack_message_2(): 上报消息确认
   - 核心函数: socket_listener(), process_server_response(), send_response()

6. esl_socket_udp_service.c
   - 路径: core/apps/esl2/ctrl/esl_socket_udp_service.c
   - 用途: UDP广播服务，设备发现和自动组网
   - 实现逻辑:
     - send_broadcast_message(): 发送UDP广播消息
       - 创建UDP socket，启用 SO_BROADCAST 选项
       - 向指定IP和端口(OPEN_BROADCAST_HOST)发送JSON格式消息
     - recevice_msg_and_send(): 接收UDP广播的线程主循环
       - 绑定本地端口接收广播消息
       - 每个接收到的消息创建独立线程处理
     - consume_message(): 消费广播消息的线程函数
       - 解析消息中的 date_type(998), master_group_id, master_ip
       - 随机sleep 1-20秒避免冲突（错峰响应）
       - 向主设备(master)发送自身设备信息（设备号+IP）
       - 更新本地 group_map 记录主设备IP
     - set_local_group_id_1/2(): 设置本地组ID
   - 核心函数: send_broadcast_message(), recevice_msg_and_send(), consume_message()

8. mqtt_subscriber.c
   - 路径: core/apps/esl2/ctrl/mqtt_subscriber.c
   - 用途: MQTT消息订阅处理，接收和处理云下发的消息
   - 实现逻辑:
     - mqtt_subscriber(): MQTT订阅回调函数，处理接收到的消息
       - 解析消息模型，获取 device_no 判断是屏1还是屏2
       - 检查 ignore 标志，如果忽略则直接上报状态
       - 根据 type 字段区分消息类型:
         - EslMessageTypeCONTENT: 内容更新
           - 检查 unbind 标志，如果为1则解绑设备（删除本地数据和UI）
           - 否则解析 content 字段:
             - esl_type==2或3: 使用新版模板引擎 parse_esl_base_model()
             - 其他: 使用旧版模板引擎 the_second_to_model()，解析 tpl_jn/tpl_pro
           - 保存数据到本地文件
           - 下载图片/视频资源
           - 异步刷新UI (lv_async_call)
           - 上报消息确认 (push_msgack_message_2)
     - 使用互斥锁保护消息处理（s_mqtt_subscriber_mutex_1/2）
   - 核心函数: mqtt_subscriber()

12. hashmap.c
    - 路径: core/apps/esl2/utils/hashmap.c
    - 用途: 哈希表实现（开放寻址法 + Robin Hood哈希）
    - 实现逻辑:
      - 基于开源实现 (Joshua J. Baker)
      - 使用开放寻址法，碰撞时采用 Robin Hood 哈希策略
      - 自动扩容: 负载因子超过60%时扩容
      - 自动缩容: 负载因子低于10%时缩容
      - 支持自定义: 哈希函数、比较函数、释放函数、分配器
      - hashmap_new(): 创建哈希表
      - hashmap_set_allocator(): 设置自定义内存分配器
      - hashmap_set_grow_by_power(): 设置扩容倍数
      - hashmap_set_load_factor(): 设置负载因子
      - 提供 sip hash 和 murmur hash 两种哈希算法
    - 核心函数: hashmap_new(), hashmap_set_allocator()

13. md5.c
    - 路径: core/apps/esl2/utils/md5.c
    - 用途: RSA Data Security, Inc. MD5消息摘要算法实现
    - 实现逻辑:
      - 标准MD5算法实现，符合 RFC 1321
      - MD5Init(): 初始化MD5上下文，设置初始向量
      - MD5Update(): 更新MD5计算
      - MD5Final(): 完成MD5计算，输出摘要
      - MD5Transform(): 对64字节块进行MD5变换（4轮，每轮16步）
      - 使用 F, G, H, I 四个非线性函数
      - 支持任意长度输入，输出128位(16字节)摘要
    - 核心函数: MD5Init(), MD5Update(), MD5Final()

================================================================================
五、整体架构总结
================================================================================

ESL2 是一个电子价签(Electronic Shelf Label)应用程序，采用 MVVM 架构:

1. Model 层 (数据模型):
   - 通过 esl_json_utils.c 解析JSON构建商品信息、模板等数据模型
   - 通过 esl_file.c 实现本地数据持久化

2. View 层 (视图):
   - 基于 LVGL v9 图形库
   - page_esl.c 是核心UI管理，负责模板渲染和内容展示
   - page_footer.c 提供状态栏（时间、网络状态）
   - qm_product_comp.c, qm_2_label.c, qm_3_label.c 提供可复用的UI组件

3. Controller 层 (控制):
   - mqtt_control.c / mqtt_subscriber.c: 通过MQTT与云服务器通信
   - esl_service.c: HTTP API调用，获取配置和内容
   - esl_socket_service.c: TCP设备间通信
   - esl_socket_udp_service.c: UDP广播实现设备发现和自动组网
   - esl_message_queue.c: OTA批量升级消息管理

4. 通信方式:
   - MQTT: 接收云端下发的价签内容更新、配置变更
   - HTTP: 查询推送服务器配置、上报状态
   - TCP: 主设备向从设备推送数据
   - UDP广播: 设备发现和自动组网

5. 特色功能:
   - 双屏支持: 支持两个独立屏幕（屏1/屏2），使用 _1/_2 后缀区分
   - 主从模式: 支持主设备和从设备组网通信
   - OTA批量升级: 通过消息队列批量处理设备升级
   - 信号量限流: 对版本更新等操作限流控制
   - 模板引擎: 支持新版(esl_base_model_t)和旧版(the_second_to_model)两种模板
   - 资源管理: 自动下载、缓存和清理图片/视频资源
